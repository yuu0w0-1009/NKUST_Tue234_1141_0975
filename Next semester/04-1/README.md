# 使用偽角度完成標準輸入的順時針排序

### 1. 核心思想：為什麼不用真實角度？
在計算座標平面上兩點的夾角時，標準做法是使用反正切函數 `atan2(dy, dx)`，它能給出精確的弧度值。然而，三角函數的底層運算非常複雜，對於需要大量且頻繁排序的圖形渲染或碰撞偵測而言，效能開銷太大。
如果我們的目的**只是為了排序**，我們根本不需要知道真實的角度是多少度，只需要一個能與真實角度保持**單調性（Monotonicity）**的數值即可。這個替代品就是「偽角度」。

### 2. 偽角度的計算原理
偽角度利用座標差值 $(dx, dy)$ 的線性比例來估算方位。程式中的公式如下：
$$p = \frac{dx}{|dx| + |dy|}$$
這個比例 $p$ 的值必定落在 $[-1, 1]$ 之間。接著，透過判斷 $dy$ 的正負號（也就是點在中心點的上方還是下方），將其映射到四個象限，形成一個連續遞增的範圍：
*  **第一、第二象限 ($dy \ge  0$)：** 返回 $1.0 - p$。數值範圍落在 $[0, 2]$。
*  **第三、第四象限 ($dy < 0$)：** 返回 $p - 1.0$。數值範圍落在 $[-2, 0)$。
最終，這會產生一個範圍落在 **$[-2, 2]$ 的週期值**。這個值在數學直角座標系中，會從正右方（3點鐘方向，值為 0）開始，**逆時針方向單調遞增**。

### 3. 如何達成「順時針」排序？
因為我們計算出來的偽角度是**逆時針遞增**的：
* 若我們將數列由小到大（遞增）排序 $\rightarrow$ 得到逆時針序列。
* 若我們將數列**由大到小（降冪）排序** $\rightarrow$ 就會得到**順時針序列**。這就是為什麼在 `compare_clockwise` 函數中，我們設定當 `a > b` 時回傳 `-1`（排在前面）。

### 4. 自訂起始方向與相對角度
為了能從任意設定的方向（起始向量 $S$）開始順時針排序，我們不能直接使用絕對的偽角度，必須計算每個點相對於起始向量的「相對距離」。
1.  **計算基準值：** 先將起始向量 $(start\_dx, start\_dy)$ 丟進公式，得到基準偽角度 $S$。
2.  **計算差值：** 對於任意待測點，算出其偽角度 $P$，並求出兩者的差值：$rel = P - S$。

### 5. 環形邊界校正 (Wrap-around Correction)
這是確保相對排序正確的最關鍵步驟。
偽角度的一圈總長度是 $4.0$（從 $-2$ 到 $2$）。當我們計算相對差值 $rel = P - S$ 時，可能會出現一個問題：由於我們是「順時針」排序，起始點應該排在最前面，其他點應該依次排在它後面。
* 如果 $rel \le  0$：表示該點在基準點的順時針方向（數值比基準小），這很合理，直接保留。
* 如果 $rel > 0$：表示該點在基準點的**逆時針方向**。在順時針繞行一圈的邏輯裡，它應該是**最後才被掃描到**的點。
為了強迫這個點排到數列的最後面，我們將它的相對值減去一整圈的長度（即 `- 4.0`）：
```c
if (rel > 0) {
rel -= 4.0;
}
```
經過這個校正後，所有的相對偽角度 $rel$ 都會變成負數或零（落在 $[-4, 0]$ 區間），接著只要統一進行**降冪排序**，就能完美呈現從起始方向開始的順時針序列。

### 6. 使用
## 範例輸入
```text
4       ---幾組待排序資料
50 50   ---參考點座標
0 1     ---參考點方向向量
56 8    ---待排序資料座標 1
1 52    ---待排序資料座標 2
93 12   ---待排序資料座標 3
39 75   ---待排序資料座標 4
```
## 範例輸出
```text
--- 順時針排序結果 ---
(93.0, 12.0) | 相對角度值: -1.469
(56.0, 8.0) | 相對角度值: -1.875
(1.0, 52.0) | 相對角度值: -3.039
(39.0, 75.0) | 相對角度值: -3.694
```



### 總結：演算法的優勢
*  **極致效能：** 完全避開了昂貴的 `atan2`、`sin`、`cos` 函數，僅依賴基礎的加減乘除與絕對值 `fabs`。
*  **尺度不變性 (Scale Invariance)：** 輸入的向量大小不影響結果。起始向量輸入 $(1, 1)$ 與 $(1000, 1000)$ 算出的基準偽角度完全相同，因為它只看比例不看長度。
*  **高穩定性：** 避開了浮點數運算中常見的角度轉換精度流失問題。
