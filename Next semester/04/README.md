
# HW04：戰車火控系統

## 題目說明
戰車會在地圖上多個預定點位之間移動，每抵達一個點位，偵察機會回傳周遭敵軍的絕對座標。受限於機載火控系統的設計，此戰車的砲塔僅能**單向順時針旋轉**來鎖定目標。戰車抵達新點位後，會以當下的車頭朝向為基準，進行一輪順時針掃蕩，依序清空該區所有目標。

### 開火順序邏輯
1.  **旋轉角度：** 依照目標相對於「此時車頭朝向」的**順時針旋轉角度**，由小到大排序 ($0^{\circ} \le \theta < 360^{\circ}$)。
2.  **優先距離：** 若多個目標處於相同的旋轉角度上，則優先攻擊**距離最近**（歐式距離 Euclidean Distance）的目標。

---

## 實作規範
1.  **環境與數值：** 程式於 Linux 環境編譯。座標 ($X, Y$) 為正整數，範圍為 $1 \le X, Y \le 500$。
2.  **輸入與結束：** * 第一行為敵軍數量 $N$ ($1 \le N \le 6$)。
    * 若讀到 $N = -1$ 則程式結束。
3.  **座標與編號：** * 讀取 $N$ 後，下一行為戰車座標 $C_x, C_y$。
    * 隨後為 $N$ 行敵軍座標 $E_x, E_y$。
    * 敵軍編號從 **0** 開始依輸入順序遞增。
    * 測資保證座標不重疊。
4.  **戰車朝向：** * **初始點位：** 預設車頭朝向正北，方向向量為 $(0, 1)$。
    * **後續點位：** 朝向為位移向量 $\vec{V} = (C_x - P_x, C_y - P_y)$，其中 $(P_x, P_y)$ 為上一個點位的座標。
5.  **輸出格式：** 於一行內輸出排序後的敵軍編號，編號間以單一空白字元分隔。

## 範例測試

### 範例輸入
```text
4
50 50
56 8
1 52
93 12
39 75
2
3 70
44 81
80 75
-1
```

### 範例輸出
```text
2 0 1 3
0 1
```

### 範例說明
* **初始回合：** 戰車位於 (50, 50)，朝向正北 (0, 1)。敵人 2 的旋轉角最小 ($131.5^{\circ}$)，依序為 2 → 0 → 1 → 3。
* **第二回合：** 戰車移至 (3, 70)，朝向向量為 $(3-50, 70-50) = (-47, 20)$。排序結果為 0 → 1。

## 提示與參考

### 解題思路
1.  **使用角度：** 利用 `atan2(y, x)` 計算徑度，平移轉換後排序。
2.  **使用偏角度 (Pseudo-angle)：** 利用座標差值計算具單調性的數值替代三角函數。
3.  **使用向量法：** 利用**向量外積 (Cross Product)** 判斷旋轉方向，無需浮點數運算。
